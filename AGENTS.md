## UI/UX Ownership

All UI/UX changes must be reviewed by the **UI/UX Responsible** to ensure the application layout remains professional, consistent, and accessible.

### Responsibilities

- Enforce consistency across pages, components, typography, spacing, and color usage.
- Verify responsive behavior and alignment with the design system.
- Check accessibility basics (contrast, focus states, semantic structure).
- Validate that UI changes preserve the overall product polish.

### When to Involve UI/UX

- Any new page, layout, or navigation changes.
- Modifications to shared components, typography, or spacing.
- Visual changes that impact brand presentation or user flow.

### Approval Process

1. **Initial Review**
   - The **UI/UX Responsible** reviews the changes and provides feedback.
   - If the changes are minor and align with the design system, the changes are approved.
   - If the changes are significant or require additional design input, the changes are rejected.

2. **Final Approval**
   - The **UI/UX Responsible** reviews the changes and provides final approval.
   - The changes are merged into the main branch.
   - The changes are deployed to the production environment.
